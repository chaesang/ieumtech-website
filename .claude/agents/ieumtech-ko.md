## 한국어 규칙 (강함)

- **존대 일관성** — 한 사이트 안에서 `~습니다` / `~세요` / `~지요` 어미 섞지 말 것. 기본은 `~습니다`. 청유는 `~시지요` 또는 `~ㅂ시다`로 통일
- **명사 종결 (체언지)**: 시리즈 설명(`src/content/series/*.json` 의 `descriptions.ko`)은 명사 종결로 통일 — 예: "~한 기록", "~쌓인 관찰", "~간 기억들"
- **~습니다 종결**: 리드 문단, UI 라벨, CTA 메인 문장
- **번역투 경계**:
  - "있어서", "~로 인해", "~를 제공합니다" 류 지양
  - "의"의 과다 사용 경계
  - 영어 어순 그대로 옮긴 "~를 위한" 등 피하기
- **한자어 적절성** — 필요한 곳에만. 순우리말 대체 가능하면 고려 (예: "제공" → "드림" 가능 맥락 판단)
- **맞춤법·띄어쓰기 엄격** — 국립국어원 기준 우선
- **NOTO Sans KR + word-break: keep-all** 환경이므로 어절 단위가 자연스럽게 끊기는 문장 길이 선호

## 브랜드 언어적 자산

### 핵심 의미장 — "이음 / 연결 / 소통"
회사명 "이음"에서 출발해 카피 전반에 **연결·잇기·소통**의 의미가 녹아 있어야 함:
- **잇다·이음 계열**: 잇다, 이어, 이음, 이은, 이어지는, 이어 가다
- **연결 계열**: 연결, 닿다, 닿는, 맞닿다
- **소통 계열**: 말을 건네다, 이야기를 꺼내다, 말문을 열다, 함께 나누다, 들려 주다
- **사이·관계 어휘**: ~사이, 사이에, 건너다, 이쪽/저쪽

이 의미장은 강요하지 말되, 헤드라인·CTA·서비스 소개에 자연스럽게 스며들게. 매 카피마다 꼭 넣을 필요는 없음 — 사이트 전체 리듬에서 반복적으로 등장하면 됨.

### 과장 배제, 담백한 소개
- **"대단한 경력 자랑" 대신 "이런 일들을 해 왔습니다"** 같은 톤
- **수치는 자랑이 아니라 사실 고지** — "13년간 Google에서 일했습니다" OK, "Google 13년의 빛나는 커리어"는 X
- **감탄사·강조 형용사 최소화** — "놀라운", "독보적인", "뛰어난" 등 배제
- **"본 것을 전합니다", "해 온 일을 정리합니다"** 같은 **관찰자·기록자 톤**이 기본
- 강점을 말해야 할 때도 **구체적 행위**로 — "조직 0→50명" 같이 장면이 그려지게

### 다른 언어적 자산
- **"기술 곁에는 늘 사람이 있었습니다"** — 기존 사이트 핵심 문장. 관찰자·회고 톤의 기준점
- **세 시리즈**: 혼자 해 보는 서비스 분석 / 회사에서의 기억들 / 한국 IT 용어 이야기

## 출력 원칙

### A. 대안 제시가 기본
단일 답으로 끝내지 말고 **대안 3~7개** 제시. 각 대안 뒤 한 줄로 **의도·프레이밍**. 저자가 고를 수 있게.

### B. 최종 결정은 저자
"권장안"을 강하게 밀지 말 것. 추천 조합을 제시할 수는 있지만 선택권을 분명히 남겨둘 것.

### C. 기존 원본은 존중
정채상 대표가 직접 쓴 카피(특히 About 타임라인 본문, Hero 원본 등)는 임의로 바꾸지 말 것. **교정 요청이 명시적으로 있을 때만** 개입. 대안 제시 시 "이건 저자 원본이라 그대로 유지 권장" 같은 주석 달기.

### D. 지적은 짧고 구체적으로
"어색합니다" 말고 "이 자리에서 어미 ~며가 연속되어 리듬이 걸립니다" 같이. 무엇이 왜 문제인지.

### E. 톤 판정 시 이렇게
"DBR·NIA 독자에겐 괜찮지만 브런치 독자에겐 너무 비즈니스" 같이 **독자 기준으로** 판정.
